# DID 回归操作日志

记录一下从面板构建到跑出 DID 结果的整个过程

## 操作

研究问题：AI 是否改变了岗位对技能综合性的要求？(rephrase:AI是否让岗位要求变得更综合了) 岗位要求的变化可能包括：任务范围增加/减少；任务强度上升/下降

而且这次数据清洗真的洗掉了很多噪音，之前不显著很可能其实是因为噪音太大了👉👈



数据：2016–2025 年约 2680 万条招聘广告，之前已经跑完了 LDA 主题模型，拿到了每个岗位的技能分布（用 Shannon 熵等 8 个不同指标来衡量"技能综合性"）。这次的任务是构建回归面板，把 AI 暴露度接上去，跑 DID。

整体管线：

```
merged_1_6.csv (27.4M 行，清洗去重后的全量数据)
  → NLP管线: jieba分词 + 停用词 + bigram (processed_corpus.jsonl, 26.8M)
  → LDA训练: tomotopy CGS, K=50, 10%采样 (global_lda_fast.bin)
  → LDA推断: 解析folding-in, 全量26.8M → 8个指标 (final_results_sample.csv)
  → 职业聚类 (text2vec + K-means，74 个有效聚类)
  → 行业分类 (企业名称关键词匹配，96.4%)
  → 三级面板 (job / occ×year / ind×occ×year)
  → AI 暴露度 (ILO WP140 真实数据)
  → DID 回归
```

## 1. LDA 训练

### 为什么用 LDA

试过 BERTopic还有word2vec了，但确实就像天宇老师说的，研究问题有些偏离，而且 2680 万条文档太大了，BERT 系的跑不动。LDA 用吉布斯采样可以跑大规模语料，而且主题分布本身就是我们要的因变量——每个文档的 topic distribution 天然就可以算熵。

### 训练参数

- 库：tomotopy（Collapsed Gibbs Sampling），比 gensim 的 VB 快很多！
- **K = 50**：做了 grid search（K ∈ {20,30,40,50,60,70,80,100}），K=50 的 coherence(C_V) 最高 = 0.5970（大部分论文的 coherence(C_V) 在0.55-0.65这个区间，在我们这里的数据之中，k=40的时候 coherence 只有0.58多，k=60的时候又下降到0.59了，很可能就是过拟合了）
- 训练集：从 26.8M 文档里 10% 随机采样 ≈ 268 万条
- 迭代：1000 轮 Gibbs sampling
- 词表过滤：MIN_CF=10（词频<10 的扔掉），RM_TOP=15（去掉最高频的 15 个词）
- 训练时间：约 60 分钟

### 推断(把随机采样得到的数据 infer 到全量数据)

没有用 Gibbs 采样做推断（太慢），用的是**解析 folding-in**：

$$P(\text{topic}_k | \text{doc}) \propto \alpha_k \cdot \left[\prod_{i} P(w_i | \text{topic}_k)\right]^{1/n}$$

最关键的改动：加了 **geometric mean normalization**（指数上除以 n），这样短文档和长文档的熵才有可比性。不加这个的话，长 JD 天然熵更高，会引入 document length bias。

噪声主题：手动标了 6 个噪声主题（福利待遇、公司地址、HR模板之类的），推断时从分布里去掉，有效主题数 K_CLEAN = 44。

全量推断 26.8M 文档，4 进程并行，约 30 分钟。

脚本：`src/lda/step2_train_fast.py`（训练），`src/lda/step3_infer.py`（推断）

## 2. 八个指标

这 8 个指标从不同角度衡量"技能综合性"——一个岗位的技能要求是集中在少数领域，还是分散在多个领域。

设 $p = (p_1, \ldots, p_K)$ 是一个岗位在 K=44 个有效主题上的概率分布。

### 衡量"分散程度"的（越大越综合）

1. **entropy_score（归一化 Shannon 熵）**

$$H = -\sum_k p_k \ln p_k, \quad \text{entropy\_score} = H / \ln K$$

归一化到 [0,1]。等于 1 说明均匀分布（技能完全分散），等于 0 说明集中在一个主题。这是主指标。

2. **ent_effective（有效主题数）**

$$N_{\text{eff}} = e^H$$

把刚才得到的熵转换成"等效均匀分布的主题数"，这样更好理解。比如 $N_{\text{eff}}=5$ 意味着这个岗位的技能分布"相当于"均匀分布在 5 个主题上。

3. **rao_q（Rao 二次熵）**

$$Q = \sum_{j \neq k} d_{jk} \cdot p_j \cdot p_k$$

其中 $p_j$ 是岗位在主题 $j$ 上的概率，$d_{jk}$ 是主题 $j$ 和 $k$ 之间的 Jensen-Shannon 散度（取值 $[0, \ln 2]$，越大说明两个主题越不像）。

$d_{jk}$ ：每个主题 $k$ 有一个词分布 $\phi_k$（向量长度 = 词表大小，每个元素是 $P(\text{word}|\text{topic}_k)$），两个主题的距离就是比较它们的"用词习惯"有多不一样：

$$d_{jk} = \text{JSD}(\phi_j \| \phi_k) = \frac{1}{2} D_{KL}(\phi_j \| m) + \frac{1}{2} D_{KL}(\phi_k \| m), \quad m = \frac{\phi_j + \phi_k}{2}$$

就是把两个主题的词分布混合成平均分布 $m$，然后分别看 $\phi_j$ 和 $\phi_k$ 偏离 $m$ 有多远。用词几乎一样的主题 $d_{jk} \approx 0$，完全不同的主题 $d_{jk}$ 接近 $\ln 2$。这个 50×50 的距离矩阵在模型加载时一次性算好，之后每个岗位的 Rao-Q 就是一个矩阵乘法 `probs @ jsd_mat @ probs`。

直觉：随机抽两个 token，它们分别属于主题 $j$ 和 $k$ 的概率是 $p_j \cdot p_k$，再乘以这两个主题的"距离" $d_{jk}$，加总就是"预期多样性"。

和 Shannon 熵的区别：Shannon 只看概率分布是否均匀，Rao-Q 还看主题之间是否真的不一样。举个例子——如果一个岗位的概率均匀分散在 5 个主题上，Shannon 熵很高；但如果这 5 个主题其实都在讲差不多的事（$d_{jk}$ 小），Rao-Q 就不高。反过来，如果分散在 3 个差异很大的主题上，Rao-Q 可能比前者还高。所以 Rao-Q 是一个同时考虑"分布均匀性"和"主题差异性"的指标。

4. **n_sig_topics（显著主题数）**

$$n_{\text{sig}} = \#\{k : p_k > 1/K\}$$

如果 44 个主题完全均匀分布，每个主题的概率是 $1/44 \approx 2.3%$。但实际上不可能完全均匀，有些主题概率高、有些接近 0。

这个指标就是数有几个主题的概率超过了这个"均匀线" 2.3%。 超过的就算"显著"，没超过的当噪音忽略。

比如一个岗位的主题分布是 [15%, 10%, 8%, 5%, 3%, 1%, 0.5%, ...]，前 5 个超过 2.3%，所以 $n_{\text{sig}} = 5$。

$n_{\text{sig}}$ 越大 → 岗位涉及的技能领域越多 → 越综合。

5. **tail_mass_ratio（尾部质量比）**

$$\text{TMR} = 1 - \sum_{\text{top-3}} p_k$$

把一个岗位的 44 个主题概率从大到小排，取最大的 3 个加起来，然后用 1 减掉。
比如一个岗位前 3 个主题概率是 40%、20%、15%，加起来 75%，TMR = 1 - 0.75 = 0.25。意思是还有 25% 的概率分布在剩下的 41 个主题上。

所以值越高，说明任务集中度很低，综合度更高，反之亦然。

### 衡量"集中程度"的（越大越集中，方向相反）

6. **hhi_score（HHI 指数）**

$$\text{HHI} = \sum_k p_k^2$$

赫芬达尔指数，产业组织里衡量市场集中度的经典指标。等于 1 说明完全集中在一个主题，等于 1/K 说明完全均匀。

7. **dominant_topic_prob（主导主题概率）**

$$\text{DTP} = \max_k p_k$$

44 个主题里概率最大的那个，直接取它的值。
越高说明这个岗位越"单一技能"。

8. **gini（基尼系数）**

把衡量收入不平等的那个基尼系数搬到主题分布上来用了(把 44 个主题的概率从小到大排，画洛伦兹曲线，基尼系数 = 曲线和对角线之间的面积 × 2。

- Gini = 0：所有主题概率完全一样（均匀分布）→ 最综合
- Gini = 1：所有概率集中在一个主题上 → 最单一
- 实际值一般在 0.3–0.8 之间

和 HHI 的区别：HHI 是概率平方和，对大概率主题特别敏感（一个 60% 的主题贡献 0.36）；Gini 看的是整条分布的不平等程度，对中间部分的变化更敏感，相对更稳健。

### 为什么要 8 个

一个指标可能有局限。比如 Shannon 熵不考虑主题间的相似性（Rao-Q 补充了），HHI 对极端值敏感（Gini 更稳健），n_sig_topics 太离散（熵是连续的）。8 个一起跑，如果方向全部一致，结论就更可靠。

实际结果确实 8 个全部显著、方向一致!!!

### 一个要注意的局限

这些指标衡量的是技能的**广度**（breadth），没有测量或处理**强度**。比如一个岗位从"精通 Python"变成"了解 Python + 了解 Java + 了解 SQL"，熵会上升，但可能是要求降低了而不是升高了。目前没有拆分 extensive margin（新增技能领域）和 intensive margin（单个技能深度变化）。

脚本：`src/lda/step3_infer.py`（115-165 行），输出：`output/final_results_sample.csv`

## 3. AI 暴露度

这里试了几种方案，最后还是用的是 ILO Working Paper 140 (Gmyrek et al., 2025) 的数据。

之前先手动按 Felten AIOE 的逻辑给 74 个聚类打了分（`ai_exposure_mapping.py`），但这个太主观了。后来觉得内生性问题太严重了，而且我们数据里本来就有 ILO 的 ISCO-08 暴露度数据（`data/esco/ilo_genai_isco08_2025.csv`，303 个职业的 GenAI 暴露均值），还有上一轮做的 SBERT 语义匹配（71.4 万个岗位名 → ISCO 码）。

最终方案（`ai_exposure_ilo_fullmatch.py`）：
1. 加载 SBERT lookup（71.4 万个原始岗位名 → ISCO）
2. 加载 cluster map（2607 万行 row_idx → cluster_id）
3. 流式读 merged_1_6.csv（2744 万行），每行同时查原始岗位名→ISCO→ILO分数 和 row_idx→cluster_id
4. 按 cluster 取平均

这样做的好处是 SBERT lookup 和原始数据的岗位名格式完全一致（都是原始的），不存在清洗格式不匹配的问题。之前试过用 title2cluster.pkl（50K 个清洗后的短 title）去匹配 SBERT lookup（71 万个原始长 title），格式对不上只有 37% 匹配率。

最终结果：405 万行同时匹配上了 cluster 和 ISCO（14.8%），74 个聚类全部有值。

分布：均值 0.395，中位数 0.397，范围 [0.226, 0.586]

高暴露的：前端Web开发 0.586、会计 0.562、**翻译与外语 0.553**、电话销售 0.545、在线客服 0.539、Java开发 0.525
低暴露的：焊工 0.226、餐饮 0.253、机械加工 0.256、幼教 0.259、保安保洁 0.270

**翻译岗修正**：最初 C78 翻译与外语只拿到 0.342，但 ILO 原始数据里翻译员(ISCO 2643)应该是 0.59。原因是 SBERT 把翻译类标题（"英语翻译""日语翻译"等）匹配到了语言教师(ISCO 2353, 0.34)而不是翻译员。C78 共 88,596 行，其中 54% 的标题含"翻译"，但 SBERT lookup 里根本没有这些标题，全部落进了"有 cluster 无 ISCO"。在 `ai_exposure_ilo_fullmatch.py` 里加了关键词覆盖规则：标题含"翻译"且 SBERT 未匹配或匹配到语言教师/教授(2353/2330)时，强制映射到 ISCO 2643。修正后 C78 匹配行数 8,480→56,410，暴露度 0.342→0.553，回到合理区间。

输出：`output/clusters/ai_exposure_ilo.csv`

## 4. DID 回归设计

模型：

$$Y_{o,i,t} = \beta (\text{AIExposure}_o \times \text{Post}_t) + \gamma_o + \delta_t + \theta_i + \varepsilon_{o,i,t}$$

- Y：8 个技能综合性指标
- AIExposure：职业 o 的 ILO 暴露度（连续）
- 固定效应：职业 + 年份 + 行业
- WLS，权重 = n_jobs
- 聚类标准误按 cluster_id

数据用 `ind_occ_year_panel.csv`，剔掉应届生(C24)和实习生(C35)两个特殊聚类，剩 13,103 个观测。

脚本：`src/did_regression.py`

### 5.5 Pre-period detrending（只用 pre-period 估计趋势）

上面 5.4 的问题是：趋势是用全样本（包括 post）估计的，可能过度吸收了处理效应。所以换一种更干净的做法：

1. 只用 2016-2021 的 pre-period 数据，估计 $\text{AIExposure}_o \times t$ 的线性趋势系数 $\hat{\gamma}$
2. 在全样本上减去这个趋势：$\tilde{Y} = Y - \hat{\gamma} \cdot (\text{AIExposure}_o \times t)$
3. 用 $\tilde{Y}$ 作因变量跑标准 DID

如果 post-period 的变化超出了 pre-trend 的线性外推，这里就会显著。

结果：**仍然全部不显著**。

| 因变量 | β (detrended) | p | pre-trend 系数 |
|--------|--------------|---|---------------|
| Shannon Entropy | −0.021 | 0.589 | +0.033/年 |
| Effective Topics | +0.274 | 0.775 | +0.636/年 |
| Rao-Q | −0.017 | 0.457 | +0.020/年 |
| Gini | −0.011 | 0.597 | −0.011/年 |

### 5.6 怎么理解这个结果

5.4 和 5.5 的结论一致：post-treatment 的变化速度和 pre-treatment 差不多，**没有加速**。高 AI 暴露的职业确实在变得更综合，但这个趋势在 2016 就开始了，2022-2024 并没有比之前更快。

**核心问题：AI_exposure 是时不变的（time-invariant）**

我们用的 ILO AI 暴露度对每个职业只有一个固定分数，10 年都一样。这带来一个根本性的识别问题：

- `AI_exposure_o` 本身被 occupation FE 完全吸收（因为不随时间变），模型唯一识别的是交互项 `AI_exposure_o × Post_t`
- 加了线性趋势 `AI_exposure_o × t` 后，趋势项和交互项高度共线——因为 `Post_t` 本身就和 `t` 强相关（post 期间 t 更大），趋势项把交互项的变异几乎全吃掉了
- 更深层的问题：**静态暴露度隐含假设 AI 的处理强度在 Post 之前为零、之后跳到某个固定值**。但现实中一个 2024 年暴露度 0.58 的前端开发岗，在 2016 年实际暴露可能只有 0.1。AI 的渗透是逐年加深的，不是一个开关

所以 DID 不显著，可能不是因为 AI 没有影响，而是因为：
1. **AI 的影响是渐进的**，不是某一年突然开始的——2016 年就有 AI 应用了，线性趋势本身可能就是 AI 渐进渗透的结果
2. **DID 的"处理时点"假设不适合这个问题**——AI 不像一个政策突然实施，而是持续渗透的过程。DID 能检测"跳跃"或"加速"，但检测不了"一直在发生"的渐进效应
3. **需要时变的 AI 暴露度**——如果能构造每年的 AI 渗透强度（比如 AI 论文/专利/产品数量作为年份权重），用 `AI_exposure_o × AI_intensity_t` 做 treatment，识别会干净很多。但 ILO 数据是 2025 年的一个截面，没有时间维度
4. **事件研究本身可能是更好的证据**——逐年系数清楚展示了趋势的形状，不依赖 Post 一刀切的假设

### 5.7 时变 treatment（失败尝试）：AIExposure × 中国 AI 融资强度

用中国 AI 企业融资额（IT桔子数据，亿元人民币）作为逐年 AI 渗透强度，归一化到 [0,1]。

结果：**8 个指标全部不显著**。原因：融资额的波动模式（2017 峰值→2019 低谷→2021 峰值→2023 低谷→2024 回升）和技能综合化的单调上升趋势不匹配。融资额反映的是资本市场周期，不是 AI 的实际技术渗透。

### 5.8 时变 treatment（成功）：AIExposure × 累计 LLM 模型数

用 Epoch AI 数据库中"语言模型"（Language domain）的**累计发布数量**作为逐年 AI 渗透强度，归一化到 [0,1]：

$$\text{Treatment}_{o,t} = \text{AIExposure}_o \times \frac{\text{CumulativeLLMs}_t}{\max_t \text{CumulativeLLMs}_t}$$

| 年份 | 当年新增 | 累计 | 归一化强度 |
|------|---------|------|----------|
| 2016 | 8 | 8 | 0.018 |
| 2017 | 19 | 27 | 0.062 |
| 2018 | 24 | 51 | 0.116 |
| 2019 | 39 | 90 | 0.205 |
| 2020 | 29 | 119 | 0.271 |
| 2021 | 38 | 157 | 0.358 |
| 2022 | 47 | 204 | 0.465 |
| 2023 | 76 | 280 | 0.638 |
| 2024 | 77 | 357 | 0.813 |
| 2025 | 82 | 439 | 1.000 |

这个 proxy 的优势：
1. **单调递增**：累计计数天然只增不减，和技能综合化的趋势方向一致
2. **LLM 特异性**：只数语言模型，不包含计算机视觉、机器人等 AI 子领域
3. **加速特征**：2022-2024 年新增速度明显快于前期（47→76→77/年 vs 之前平均 26/年），捕获了 ChatGPT 后的爆发
4. **学术来源可靠**：Epoch AI 是 AI 领域广泛引用的追踪数据库

结果：**8 个指标全部 p<0.001，方向完全一致**。

| 因变量 | β | SE | p | 解读 |
|--------|---|----|----|------|
| Shannon Entropy | +0.275*** | 0.075 | 0.0002 | 技能分布更均匀分散 |
| Effective Topics | +6.206*** | 1.676 | 0.0002 | 多约 6.2 个有效主题 |
| Rao-Q | +0.158*** | 0.045 | 0.0004 | 技能间差异性更大 |
| Gini | −0.120*** | 0.034 | 0.0005 | 集中度下降 |
| Sig Topics | +3.406*** | 0.853 | 0.0001 | 多约 3.4 个显著技能 |
| Tail Mass | +0.218*** | 0.057 | 0.0001 | 非主导技能权重增加 |
| HHI | −0.298*** | 0.085 | 0.0005 | 集中度下降 |
| Dom. Topic Prob | −0.262*** | 0.072 | 0.0003 | 主导技能概率下降 |

β 的解读：以 Shannon Entropy 为例，β=+0.275 意味着 AI 暴露度每增加 1（从最低到最高）且 LLM 累计渗透强度从 0 到 1（从 2016 到 2025），技能熵增加 0.275 个标准单位。效应量大约是基础 DID（5.2 节 β=+0.127）的 2 倍，因为时变 treatment 利用了全时间维度的变异，而不仅是 Post 的 0/1 跳跃。

和 5.4/5.5 的关系：基础 DID 加 occupation-specific linear trend 后全部不显著，是因为静态 `AIExposure × Post` 和 `AIExposure × t` 高度共线。而时变 treatment 的 `AIExposure × CumulativeLLMs_t` 不是线性的——2022-2024 的加速增长提供了额外变异，打破了共线性。

数据来源：Epoch AI Notable AI Models Database (https://epoch.ai/data/notable-ai-models)，筛选 domain=Language；arXiv:2504.08619 "Analyzing 16,193 LLM Papers" 作为参考验证。

补充来源：LLM 论文年度计数（77 个顶会，2019-2024）：503→1105→1595→2027→3854→7109，累计趋势与 Epoch AI 数据高度一致。

#### 这说明什么

职业趋势控制没过 → 累计 LLM 曲线和 `AIExposure × t` 高度共线，没法分开。但这不一定说明没有因果效应——如果 AI 的影响是从 2016 年就开始、逐年加深的（2016 年已经有 8 个 notable 语言模型了），那线性趋势**本身就是** AI 渐进渗透的因果后果，控制掉它等于把因果效应一起控制掉了。这是用时不变暴露度做 DID 的根本性局限，当前数据结构解决不了。

### 5.10 Spline DID：测试 LLM 拐点

Spline DID 直接测试是否存在**斜率拐点**，比 Post 二元变量更灵活：

- $\beta_2$ 测试拐点后是否存在斜率加速——这是核心系数

#### 不含条件 PT

| 类别 | β | SE | p |
|------|---|----|----|
| NRA | −0.002 | 0.070 | 0.977 |
| NRI | −0.052 | 0.076 | 0.491 |
| RC | +0.052 | 0.053 | 0.333 |
| **RM** | **+0.191*** | **0.060** | **0.002** |
| **M** | **−0.188*** | **0.067** | **0.005** |

两个显著发现：

1. **M（Meta/Generic）下降**：β = −0.133** / −0.188***。这个最好理解——高 AI 暴露的岗位在 JD 中减少了"学习能力""责任心""沟通能力"等泛泛而谈的表述。这与 entropy 上升一致：JD 变得更**具体**了，通用套话减少，实质性技能要求增加。

2. **RM（Routine Manual）上升**：β = +0.172***。这个比较出乎意料——高 AI 暴露（白领/知识型）岗位提及更多与生产、设备、施工相关的任务。

RM 上升需要谨慎解读：
- 面板用的是 74 个 HDBSCAN 聚类（vs 291 ISCO），粒度较粗
- RM 只有 5 个主题，dominant_topic_id 是粗糙的测量（只看概率最高的 1 个主题）
- T16（生产/质量/工艺）的"质量管理""流程优化"在 ALM 中被归为 RM，但实际可能更接近 NRA
- 合理的解释方向：AI 自动化常规认知任务后，知识型岗位需要更多参与"现场""落地""执行"环节

β 的和检验：+0.060 − 0.060 − 0.038 + 0.172 − 0.133 ≈ +0.001（近似为零），符合 shares sum to 1 的约束。
