# Step 1：从招聘广告中抽取技能/任务表达

## 背景

我们在做一篇实证论文，研究"生成式AI是否推动了岗位技能需求综合化"。核心方法是在招聘广告文本上训练LDA主题模型，用主题分布的Shannon熵衡量技能综合性，然后跑岗位层DID回归。

当前问题：LDA在半清洗的JD文本上训练，学到的主题更像"职业类别"（医疗护理、电商运营、法律事务）而非"技能/任务维度"（数据分析、项目管理、沟通协调）。根源在于输入文本包含大量职业锚定信息（行业名词、学历要求、经验年限、福利描述），职业信号太强，LDA无论K取多少都优先捕捉职业聚类。

本步骤的目标：对现有`cleaned_requirements`列做**句级过滤**，只保留技能/任务表达，剔除学历、经验、年龄、福利、营销等非技能句。输出一份新的预处理语料，供后续LDA训练使用。

## 输入

| 列名 | 类型 | 说明 |
|------|------|------|
| `企业名称` | str | |
| `招聘岗位` | str | 职位名称 |
| `工作城市` | str | |
| `工作区域` | str | |
| `最低月薪` | float | |
| `最高月薪` | float | |

## 处理逻辑

### 1. 切句

对每条`cleaned_requirements`，按以下分隔符切成句子：
- 句号（。）
- 分号（；或;）
- 换行符（\n）
- 编号模式作为新句起点：
  - 阿拉伯数字编号：`1、` `2.` `2．` `3、`
  - 括号编号：`（1）` `(1)` `（一）`
  - 中文编号：`一、` `二、` `三、`
- **多空格分隔**：连续2个以上空格视为分隔符（实际数据中常见条目间用空格而非标点分隔）

切句后，丢弃长度≤3个字符的片段（过短无意义）。

### 2. 逐句分类：保留 or 剔除

**剔除规则（满足任一则整句删除）**：

| 类别 | 关键词/模式 | 示例 |
|------|------------|------|
| 学历句 | 本科、大专、硕士、博士、学历、毕业、全日制、统招 | "本科及以上学历" |
| 经验年限句 | X年以上（含中文数字：一二三四五六七八九十）、工作经验、从业经验、年以上经验、年工作经历、工作经历 | "3年以上相关工作经验"、"三年及以上营销经验" |
| 年龄外貌句 | 岁、年龄、身高、形象、气质 | "22-35岁，形象气质佳" |
| 福利薪资句 | 五险、社保、公积金、年终奖、带薪、底薪、提成、补贴、住宿、食宿、工作餐 | "五险一金+带薪年假" |
| 招聘营销句 | 直招、非中介、详见官网、投递、面试、简历 | "本公司直招，非中介机构" |
| 纯软性人格句 | 整句不含任何保留规则关键词，且主要由以下词构成：吃苦耐劳、责任心、抗压、积极主动、热爱、认真负责、团队精神、为人正直、服从安排、沟通能力强、学习能力强、较强的沟通能力、团队合作、善于沟通。**判定方法**：先检查保留规则，如果都不命中，再检查是否含软性人格词，若含则剔除 | "吃苦耐劳，有责任心"、"有较强的沟通能力" |
| 政治/身份条件 | 党员、政治面貌、户籍、男女不限、性别 | "中共党员优先" |

**保留规则（满足任一则保留）**：

| 类别 | 关键词/模式 | 示例 |
|------|------------|------|
| 技能动词句 | 熟悉、掌握、精通、熟练、使用、运用、操作、了解（后接技术对象时） | "熟练使用Python和SQL" |
| 任务动词句 | 负责、开发、设计、编写、分析、管理、维护、搭建、策划、撰写、制定、推动、跟进、对接、测试、调试、部署 | "负责后端API开发与维护" |
| 工具/技术句 | 含具体工具、软件、技术栈、框架、语言、平台名称 | "熟悉Spring Boot框架" |
| ESCO词典命中 | 句中有词命中ESCO技能词典（`/Users/yu/code/code2601/TY/data/esco/jieba_dict/esco_jieba_dict.txt`） | — |

**分类优先级（三步判定）**：

1. **先查剔除规则**（学历、经验、年龄、福利、营销、政治/身份）：命中任一则剔除，不再往下判
2. **再查保留规则**（技能动词、任务动词、工具/技术、ESCO命中）：命中任一则保留
3. **都不命中→查软性人格词**：含软性人格词则剔除；不含则**丢弃**（既无技能信号也无明确噪音的句子，默认不保留，宁缺毋滥）

**注意**：如果一句话同时含学历词和技能词（如"本科学历，熟练使用CAD"），按当前逻辑整句剔除。这会造成少量误删，但比误保留非技能句的风险更小。后续可优化为子句级别切分。

### 3. 重组文档

对每条JD，将所有保留的句子拼接为一个新文档`skill_text`。

### 4. 质量标记

| 字段 | 定义 |
|------|------|
| `skill_text` | 过滤后拼接的技能/任务文本 |
| `has_skill_text` | 1 if skill_text非空且字符数≥10，else 0（字符数判定，不需要分词） |
| `skill_char_count` | skill_text的字符数 |
| `original_char_count` | 原始cleaned_requirements的字符数 |
| `n_sentences_total` | 切句后总句数 |
| `n_sentences_kept` | 保留的句数 |

## 输出

**输出文件**：`/Users/yu/code/code2601/TY/output/skill_filtered_corpus.csv`

每行一条JD，包含以下列（至少）：
- `job_id`（自生成，格式 `{window}_{行号}`，如 `w2022_0`）
- `year`（int，从`招聘发布年份`转换）
- `skill_text`
- `has_skill_text`
- `skill_char_count`
- `original_char_count`
- `n_sentences_total`
- `n_sentences_kept`

同时输出一份统计摘要到stdout或日志，包含：
- 总JD数
- has_skill_text=1 的比例
- skill_char_count 的均值/中位数/分位数
- n_sentences_kept / n_sentences_total 的均值
- 按year分组的上述统计
